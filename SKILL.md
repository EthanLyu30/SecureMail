## 核心约束（必须遵守）

### 1. 技术约束
- **禁止使用成熟开源邮箱系统库**（如Dovecot, Postfix, Courier等）作为核心实现
- 可以使用通用网络库（socket, asyncio）和基础加密库（hashlib, cryptography）
- 必须说明每个使用的库/工具的用途

### 2. 功能约束
- 必须实现server进程和client进程两个独立进程
- 两个域名服务器之间必须可以互相发信
- 存储必须逻辑隔离（不可直接读写对方用户数据目录）

## 协议设计要点

### 消息格式 (JSON over TCP)
```json
{
  "type": "命令类型",
  "seq": 序列号,
  "from": "发送者",
  "to": "接收者",
  "body": {
    "命令参数"
  },
  "signature": "数字签名（可选）"
}
```

### 认证流程
1. 客户端连接服务器
2. 客户端发送LOGIN请求（带用户名和加密密码）
3. 服务器验证，返回SESSION_TOKEN
4. 后续请求携带SESSION_TOKEN
